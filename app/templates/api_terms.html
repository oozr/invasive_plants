Data Limitations

The API reports records found in the current dataset. Plant taxonomy, common names, jurisdiction boundaries, legal classifications, source URLs, and regulatory materials can change and may be incomplete, delayed, ambiguous, or outdated.

The API may produce false positives, false negatives, ambiguous matches, incomplete source coverage, or stale results.

API Keys And Security

API keys are confidential credentials. Customers must keep keys secure, limit access to authorized systems and personnel, and must not share keys with third parties or embed them in public client-side code.

Use Restrictions

Enterprise API responses may be used for approved operational workflows such as catalog review, checkout controls, fulfillment review, restricted-shipping logic, seller support, customer-support records, and compliance audit records.

API responses must not be used to reconstruct, mirror, bulk replicate, or create a substitute for the Regulated Plants Database or a competing plant regulatory data product.

Privacy And Request Data

The API is designed for plant queries and destination jurisdiction fields such as country and region/state/province. Customers should not submit customer names, street addresses, email addresses, phone numbers, payment information, or other unnecessary personal data.

See the Privacy notice for information about account, contact, and API administration data.
