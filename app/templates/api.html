Regulated Plants API

A query service for checking whether a plant is regulated in a destination jurisdiction.

Built from the Regulated Plants Database, the API supports checkout, catalog review, and marketplace compliance workflows where a plant name must be compared against a ship-to location.

Primary Use Case

Given a plant name and a ship-to jurisdiction, return whether the species has a regulation record in the current dataset.

Access Model

API usage is token protected. Approved organizations receive a bearer token after review; paid subscription and metered billing will be connected later.

Designed for cautious regulatory checks

The endpoint is intentionally conservative. It reports whether a matched species is regulated in a requested jurisdiction according to the current release, and returns authority details, source URLs, raw classifications, notes, and release metadata where available.

It does not say a plant is legally allowed, banned, or prohibited to buy. If a common name is ambiguous or misspelled beyond a safe threshold, the API asks the user to choose from suggested species before returning a regulatory result.

POST /v1/regulatory-check
{
  "plant_query": "Cortaderia jubata",
  "ship_to": {
    "country": "US",
    "region": "CA"
  }
}
