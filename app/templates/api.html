API

Two API tracks: the current website API and a future external US compliance API.

Website API Active

API A: Global website API

This API powers the public Regulated Plants website. It is global, supports exploratory use, and can evolve quickly to support website features and research communication.

Scope: website UX and public-facing product functionality.

Current Website API Scope

Global map + jurisdiction data

Supports site map rendering, jurisdiction overlays, and public data exploration workflows.

  • Global jurisdiction coverage
  • Regional, national, and international toggles
  • Optimized for website interaction patterns

Species search + species detail

Supports scientific/common name discovery and jurisdiction-linked species navigation on the website.

  • Search and detail routes for the site UI
  • Global dataset context
  • Flexible response shape for front-end iteration

Next Track (Planned)

API B: US external compliance API

A separate API will be released for external partner integrations. Initial scope is US-only, with stricter stability, authentication, and versioning requirements.

Partner-focused characteristics

Smaller endpoint surface, cleaner contract, stable versioning, and implementation-grade documentation.

Shared core engine

Both APIs will run on the same core data and compliance logic to avoid duplicate business rules.
