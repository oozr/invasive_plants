Coming Soon

Built for trusted operational use

We are preparing an API release for teams that need machine-readable regulated plant data in production systems. The initial focus is on helping marketplaces, retail platforms, and seller operations teams identify listings that may conflict with jurisdiction regulations before they create downstream compliance risk.

This release is being shaped around trust, clarity, and integration readiness.

Two Core Benefits

Full set of jurisdiction regulations

Access regulation coverage across regional, national, and international layers in a machine-readable format. This is intended to support compliance screening, policy checks, and jurisdiction-aware decision workflows.

  • Regional, national, and international scope
  • Structured outputs for platform and compliance systems
  • Clear jurisdiction context for downstream operational rules

Full set of species data, including common names

Retrieve the species data needed for matching and review workflows, including scientific names, common names, and related taxonomic context. This is especially important for seller-submitted product titles and catalog data.

  • Scientific names and common names
  • Cleaner matching for catalog and seller content
  • More reliable handling of plant identity in operational checks

Planned Release Priorities

Professional documentation

The API release will include professional documentation with endpoint references, response examples, schema notes, and implementation guidance suitable for engineering and compliance teams.

Stable and integration-ready

We intend to ship versioned endpoints, predictable response structures, and clear change communication so teams can integrate with confidence.

Traceable and reviewable

Source provenance, update visibility, and transparent data handling are core to the release plan, because trust matters when the API is used in compliance-sensitive workflows.

Potential Use Cases

  • Pre-listing checks for products that may be restricted in specific jurisdictions
  • Seller guidance and onboarding workflows for regulated plant categories
  • Catalog screening and exception review by compliance teams
  • Jurisdiction-aware policy checks inside marketplace operations
  • Internal tooling for trust, safety, and regulatory review teams
